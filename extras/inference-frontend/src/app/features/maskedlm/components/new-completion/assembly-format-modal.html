Assembly Format

The assembly you enter here must be in Undertale pretoken format. This is a bit different from most assembly syntax that you may be familiar with (Intel, AT&T, GAS).

Different How?

Undertale's pretoken format consists of only letters a-zA-Z, numbers 0-9, some special characters [], and spaces.

In general, you should remove commas, treat memory reference brackets [] as their own token, and convert immediates as well as symbol references (variables, function name references) to their integer equivalent.

Remove Commas
xor rax, rax
xor rax rax
Memory Reference Syntax
mov rax, qword ptr [rsi+2h]
mov rax qword [ rsi + 2 ]
Immediates
mov rax, 2ah
mov rax 42
Addresses
call function_label
call 3735928559
Why?

Our assembly preprocessing pipeline includes logic that requires disassembler internals and access to the binary form of the assembly. We didn't want to have to assemble and then disassemble your input just to get inference results.

What does [UNK] mean?

[UNK] is the unknown token. If the model has no way to represent a given input, then it emits [UNK] tokens, typically one for each character of an unrecognized input.

This most commonly happens when you feed the model an input which is not properly formatted disassembly pretokens. But it can happen for legitimate assembly because of idiosyncrasies in the underlying disassembler.

For example, Binary Ninja never emits the jnz instruction, instead only emitting jne. jnz is therefore not a part of the model's vocabulary, even though it is a valid instruction, and the model will emit [UNK] [UNK] [UNK].

We don't have a comprehensive list of these edge cases yet (they were learned from our dataset) but we could use your help in finding them! Please provide detailed feedback on completions.
